## Пользовательские типы данных
В С++ можно использовать свои типы данных а не опираться на уже существующие: int, double, char и тд.

Пользовательские типы могут быть такими:

1. Структуры - просто коробка с полями. Это группа переменных под одним именем, похоже на объект.
```c++
struct Worker {
    char fio[30];
    int date;
    int code;
    double salary;
};

// как объект
// Worker = {  
// имя,  
// дата,  
// код,  
// зарплата  
// }

// 👉 Использование:
Worker w;
w.salary = 50000;
```

2. union - одна память, разные интерпретации. Один и тот же участок памяти как разные типы.
```c++
union Data {
    int i;
    float f;
};
```
👉 Что это значит:
- `i` и `f` занимают **одну и ту же память**
- ты можешь использовать **либо одно, либо другое**
```c++
Data d;
d.i = 10;
d.f = 3.14; // перезапишет i
```
📌 Используется редко, обычно для оптимизации или низкоуровневого кода

3. Битовые поля - почти никогда не нужны новичку. Доступ к отдельным битам.
```c++
struct Flags {  
unsigned int a : 1;  
unsigned int b : 1;  
};
```
👉 Это значит:
- `a` и `b` занимают **по 1 биту**
📌 Честно:  
👉 Можешь пока **забыть про это** — это уже продвинутый уровень

4. enum - список значений.
Набор именованных целых констант.
```c++
enum Color { Red, Green, Blue };
```
это превращается в
```c++
Red = 0
Green = 1
Blue = 2
```
🔥 Более современный вариант (как у тебя в примере)
```c++
enum class Operation { Add, Subtract, Multiply };
```


`typedef` — просто «псевдоним типа». Он позволяет переименовать стандартный тип на то, что мы хотим.
```c++
typedef unsigned int UINT;

UINT a; // вместо unsigned int a;
```


# Константы и переменные
🔹 `#define` vs `const`
```c++
#define MY_PI 3.14

const double PI = 3.14;

// ======================

#define A 10+5  
int x = A * 2;

// 👉 Получится:

const int A = 10 + 5;

```

# Оператора сдвига

# 🔸 Сдвиг влево `<<`

👉 Смысл:  
**сдвигаем все биты влево, справа добавляются нули**
```c++
int x = 5;      // 00000101
x = x << 1;     // ?

// 00000101
// → 00001010
```
👉 Это уже: 10 а не 5
## 🧠 Важное правило
👉 `x << 1` = умножить на 2  
👉 `x << 2` = умножить на 4  
👉 `x << 3` = умножить на 8

📌 Ещё пример
```c++
int x = 3;   // 00000011
x = x << 2;

// 00000011 (3)
// 00001100 (12)
```


также работает сдвиг вправо.
## 🧠 Правило
👉 `x >> 1` = деление на 2  
👉 `x >> 2` = деление на 4
```c++
int x = 8; // 00001000  
x = x >> 1;

// 00001000
// → 00000100
```

🔥 Разберём пример
```c++
unsigned int value = 52;
value <<= 1;

// переводим 52 в двоичную
// 52 = 00110100

// сдвигаем влево на 1
// 00110100
// → 01101000
```
📌 Это число 104
```c++
cout << value; // 104 получилось
```

# 🔹 В чём проблема с `cin`
При работе с cin вводится только то, что до пробела, символа табуляции или перевода строки.
```c++
string s;
cin >> s;

// Ввод: Hello world
// Вывод: Hello
```

# 🔸 Что делает `getline`
```c
cin.getline(s, 80);
```
он читает всю строку целиком до enter.
👉 `80` — это:
> максимальная длина строки

# 🧠 Как запомнить
👉 `cin >>` → «одно слово»  
👉 `getline` → «вся строка»

💡 Современный вариант (лучше)
```c
string s;
getline(cin, s);
```
👉 Работает удобнее, чем `char s[80]`

# Тернарная операция
операнд1 ? операнд2 : операнд3

max = a <= b ? b : a;
Здесь переменной max присваивается максимальное значение из переменных а и b.

a < b ? a: b = c * x + d;
Здесь значение выражения c * x + d присваивается меньшей из двух переменных а и b.
